# 現基本給設定タブ 要件定義

## 概要

号俸テーブルを設計する際に、従業員の「現在実際に支払っている基本給」を入力するための機能を追加する。

### 背景・目的

- 号俸テーブルの「従業員当てはめ」機能では、従業員の基本給と号俸テーブルを照合している
- 現状、従業員の`baseSalary`フィールドは号俸テーブルから逆算した値が入っている
- しかし、号俸テーブルを新規設計・調整する際には、**実際に今支払っている基本給**を知る必要がある
- この「現基本給」は従業員マスタとは別で、**号俸テーブル設計のための一時的なデータ**として管理する

## 機能要件

### 1. UIの配置

- **場所**: 号俸テーブル設定ページ (`/companies/[companyId]/salary-table`)
- **タブ追加**: 既存タブ「設定」「テーブル」「改定基準」の右側に「現基本給設定」タブを追加

```
[設定] [テーブル] [改定基準] [現基本給設定]  ← 新規追加
```

### 2. 現基本給設定タブの画面構成

#### 2.1 ヘッダー部分
- タイトル: 「現基本給設定」
- 説明文: 「号俸テーブル設計のため、従業員の現在の基本給を入力してください」
- 一括保存ボタン（右上）

#### 2.2 従業員リスト（テーブル形式）

| 列名 | 表示内容 | 編集可否 |
|------|----------|----------|
| 社員番号 | `employee.employeeNumber` | 表示のみ |
| 氏名 | `lastName firstName` | 表示のみ |
| 等級 | `grade.name`（正①〜正⑥など） | 表示のみ |
| 職種 | `jobType.name` | 表示のみ |
| 現基本給 | 入力フィールド（数値、3桁カンマ表示） | **編集可** |

#### 2.3 機能
- **ソート**: 等級順（デフォルト）、社員番号順、氏名順
- **フィルター**: 等級でフィルタ可能
- **一括保存**: 変更した全データを一括で保存
- **入力バリデーション**: 0以上の整数のみ許可

### 3. データモデル

#### 3.1 新規テーブル: `EmployeeCurrentSalary`

号俸テーブル設計用の現基本給を保存するテーブル。

```prisma
model EmployeeCurrentSalary {
  id            String   @id @default(cuid())
  employeeId    String
  salaryTableId String
  currentSalary Int      // 現在実際に支払っている基本給
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  employee    Employee   @relation(fields: [employeeId], references: [id], onDelete: Cascade)
  salaryTable SalaryTable @relation(fields: [salaryTableId], references: [id], onDelete: Cascade)

  @@unique([employeeId, salaryTableId])
  @@map("employee_current_salaries")
}
```

**設計ポイント**:
- `salaryTableId`で紐づけ: 号俸テーブルごとに現基本給を保存できる
- 従業員マスタの`baseSalary`とは独立して管理
- 号俸テーブルが削除されたら、関連する現基本給データも削除（Cascade）

### 4. API設計

#### 4.1 現基本給一覧取得

```
GET /api/salary-tables/[id]/current-salaries
```

**レスポンス例**:
```json
{
  "employees": [
    {
      "employeeId": "emp001",
      "employeeNumber": "EMP001",
      "name": "田中 太郎",
      "gradeName": "正④",
      "jobTypeName": "多能工",
      "currentSalary": 280000
    }
  ]
}
```

#### 4.2 現基本給一括保存

```
PUT /api/salary-tables/[id]/current-salaries
```

**リクエストボディ**:
```json
{
  "salaries": [
    { "employeeId": "emp001", "currentSalary": 280000 },
    { "employeeId": "emp002", "currentSalary": 420000 }
  ]
}
```

### 5. 「従業員当てはめ」との連携

現基本給が設定されている場合、「従業員当てはめ」タブでは：

1. **優先順位**:
   - `EmployeeCurrentSalary.currentSalary` が存在すればそれを使用
   - 存在しなければ従来通り `Employee.baseSalary` を使用

2. **表示の区別**:
   - 現基本給が設定済み: 通常表示
   - 現基本給が未設定: グレーアウト or 「未入力」ラベル表示

### 6. UI実装詳細

#### 6.1 コンポーネント構成

```
src/components/salary-table/
├── CurrentSalaryTab.tsx        // タブ全体のコンテナ
├── CurrentSalaryTable.tsx      // テーブル表示
└── CurrentSalaryInput.tsx      // 入力フィールド（3桁カンマ対応）
```

#### 6.2 入力フィールドの仕様

- `type="text"` + `inputMode="numeric"`
- 3桁カンマ区切りで表示
- 入力時はカンマを除去して数値として保存
- 「円」の単位を右側に表示
- 空欄 or 0 の場合は未入力扱い

#### 6.3 保存時の挙動

- 「保存」ボタンクリックで一括保存
- 未変更のデータはスキップ（差分更新）
- 保存成功時はトースト通知
- エラー時はエラーメッセージ表示

### 7. 画面遷移・UXフロー

```
1. 号俸テーブル設定ページを開く
2. 「現基本給設定」タブをクリック
3. 従業員リストが表示される（初期値は空 or 0）
4. 各従業員の現基本給を入力
5. 「保存」ボタンをクリック
6. 「テーブル」タブや「従業員当てはめ」で確認
```

## 非機能要件

### パフォーマンス
- 従業員100名程度を想定
- ページネーションは不要（全件表示）
- 入力時のデバウンス処理は不要（保存時に一括送信）

### データ整合性
- 従業員が削除されたら、現基本給データも削除
- 号俸テーブルが削除されたら、現基本給データも削除
