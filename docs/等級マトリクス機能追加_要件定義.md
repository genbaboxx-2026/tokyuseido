# 等級×職種マトリクス 機能追加 要件定義

## 概要

等級制度管理ページ（`/companies/[companyId]/grades`）の「有効/無効設定」タブにある等級×職種マトリクスに、以下の機能を追加する。

1. **等級削除機能**
2. **職種大分類単位の一括チェック機能**

## 追加機能1: 等級削除

### 要件

- 各等級行に「削除」ボタン（またはアイコン）を追加
- 削除ボタンをクリックすると確認ダイアログを表示
- 確認後、等級を削除

### 削除時の制約

以下の場合は削除を禁止し、エラーメッセージを表示：

| 制約 | エラーメッセージ |
|------|------------------|
| 該当等級に従業員が紐づいている | 「この等級には従業員が紐づいているため削除できません」 |
| 該当等級に評価データが存在する | 「この等級には評価データが存在するため削除できません」 |

### 動作仕様

#### 列ヘッダー（職種大分類）のチェックボックス

| 操作 | 動作 |
|------|------|
| チェックON | その大分類に属する全職種の**全等級**をONにする |
| チェックOFF | その大分類に属する全職種の**全等級**をOFFにする |
| 状態表示 | 全てON → ✓、一部ON → ▣（中間状態）、全てOFF → ☐ |

| 操作 | 動作 |
|------|------|
| チェックON | その等級の**全職種**をONにする |
| チェックOFF | その等級の**全職種**をOFFにする |
| 状態表示 | 全てON → ✓、一部ON → ▣（中間状態）、全てOFF → ☐ |

### 等級削除API

**エンドポイント**: `DELETE /api/grades/[gradeId]`

**処理フロー**:

```typescript
export async function DELETE(request: NextRequest, context: { params: { gradeId: string } }) {
  const { gradeId } = context.params;

  // 1. 認証チェック
  const session = await auth();
  if (!session?.user) {
    return NextResponse.json({ error: "認証が必要です" }, { status: 401 });
  }

  // 2. 等級の存在確認
  const grade = await prisma.grade.findUnique({
    where: { id: gradeId },
    include: {
      _count: {
        select: {
          employees: true,
        },
      },
    },
  });

  if (!grade) {
    return NextResponse.json({ error: "等級が見つかりません" }, { status: 404 });
  }

  // 3. 従業員が紐づいているか確認
  if (grade._count.employees > 0) {
    return NextResponse.json(
      { error: "この等級には従業員が紐づいているため削除できません" },
      { status: 400 }
    );
  }

  // 4. 評価データが存在するか確認（必要に応じて）
  const evaluationCount = await prisma.employeeEvaluation.count({
    where: {
      employee: {
        gradeId: gradeId,
      },
    },
  });

  if (evaluationCount > 0) {
    return NextResponse.json(
      { error: "この等級には評価データが存在するため削除できません" },
      { status: 400 }
    );
  }

  // 5. トランザクションで削除
  await prisma.$transaction(async (tx) => {
    // GradeJobTypeConfig は onDelete: Cascade で自動削除
    // GradeRole も onDelete: Cascade で自動削除
    await tx.grade.delete({
      where: { id: gradeId },
    });
  });

  return NextResponse.json({ success: true });
}
```

#### 削除ハンドラー

```typescript
const handleDeleteGrade = async () => {
  if (!deleteTargetGrade) return;

  setIsDeleting(true);
  setDeleteError(null);

  try {
    const response = await fetch(`/api/grades/${deleteTargetGrade.id}`, {
      method: "DELETE",
    });

    if (!response.ok) {
      const data = await response.json();
      throw new Error(data.error || "等級の削除に失敗しました");
    }

    setDeleteTargetGrade(null);
    router.refresh();
  } catch (err) {
    setDeleteError(err instanceof Error ? err.message : "等級の削除に失敗しました");
  } finally {
    setIsDeleting(false);
  }
};
```

#### 一括チェックのロジック

```typescript
// 職種大分類単位の一括チェック
const handleCategoryToggle = (categoryId: string, newValue: boolean) => {
  const category = jobCategories.find((c) => c.id === categoryId);
  if (!category) return;

  const jobTypeIds = category.jobTypes.map((jt) => jt.id);

  setPendingChanges((prev) => {
    const next = new Map(prev);

    matrix.forEach((row) => {
      jobTypeIds.forEach((jobTypeId) => {
        const key = `${row.grade.id}-${jobTypeId}`;
        const originalValue = row.jobTypes.find((c) => c.jobType.id === jobTypeId)?.isEnabled ?? false;

        if (newValue === originalValue) {
          next.delete(key);
        } else {
          next.set(key, newValue);
        }
      });
    });

    return next;
  });
};

// 等級（行）単位の一括チェック
const handleRowToggle = (gradeId: string, newValue: boolean) => {
  setPendingChanges((prev) => {
    const next = new Map(prev);
    const row = matrix.find((r) => r.grade.id === gradeId);

    if (!row) return prev;

    row.jobTypes.forEach((cell) => {
      const key = `${gradeId}-${cell.jobType.id}`;
      if (newValue === cell.isEnabled) {
        next.delete(key);
      } else {
        next.set(key, newValue);
      }
    });

    return next;
  });
};

// チェックボックスの状態を計算（全ON、一部ON、全OFF）
const getCategoryCheckState = (categoryId: string): "checked" | "indeterminate" | "unchecked" => {
  const category = jobCategories.find((c) => c.id === categoryId);
  if (!category) return "unchecked";

  const jobTypeIds = category.jobTypes.map((jt) => jt.id);
  let onCount = 0;
  let totalCount = 0;

  matrix.forEach((row) => {
    jobTypeIds.forEach((jobTypeId) => {
      const isEnabled = getCellValue(row.grade.id, jobTypeId, 
        row.jobTypes.find((c) => c.jobType.id === jobTypeId)?.isEnabled ?? false
      );
      if (isEnabled) onCount++;
      totalCount++;
    });
  });

  if (onCount === 0) return "unchecked";
  if (onCount === totalCount) return "checked";
  return "indeterminate";
};

const getRowCheckState = (gradeId: string): "checked" | "indeterminate" | "unchecked" => {
  const row = matrix.find((r) => r.grade.id === gradeId);
  if (!row) return "unchecked";

  let onCount = 0;
  const totalCount = row.jobTypes.length;

  row.jobTypes.forEach((cell) => {
    const isEnabled = getCellValue(gradeId, cell.jobType.id, cell.isEnabled);
    if (isEnabled) onCount++;
  });

  if (onCount === 0) return "unchecked";
  if (onCount === totalCount) return "checked";
  return "indeterminate";
};
```

## 追加機能3: 役割責任モーダルの保存ボタン統一

### 要件

- 現状: 各等級（正1、正2...）ごとに「保存」ボタンがある
- 変更後: モーダルのヘッダーに**1つの保存ボタン**だけを表示

### UI変更

### 動作仕様

1. モーダルを開いたときに全等級の役割責任を取得
2. 各等級の内容を編集可能（追加/編集/削除）
3. 変更を追跡し、変更があれば「保存」ボタンを有効化
4. 「保存」ボタンをクリックすると、変更のある等級の役割責任を一括保存

## 注意事項

1. **カスケード削除**: GradeJobTypeConfig、GradeRole は `onDelete: Cascade` が設定されているので、等級削除時に自動削除される
2. **保存のタイミング**: 一括チェック操作も「保存」ボタンを押すまでDBには反映されない（pendingChanges で管理）
3. **トランザクション**: マトリクス更新APIは現在トランザクションを使用していないが、可能であればトランザクション化を推奨
