# 号俸テーブル設定 - 統一保存ボタン 要件定義

## 概要

号俸テーブル設定ページの4つのタブ（現基本給設定、詳細設定、テーブル、改定基準）で、個別に存在する保存ボタンを1つの統一保存ボタンに集約する。

## 現状の構造

### ファイル構成

```
src/
├── app/companies/[companyId]/salary-table/page.tsx  # 親コンポーネント
└── components/salary-table/
    ├── CurrentSalaryTab.tsx          # 現基本給設定タブ
    ├── SalaryTableInputPanel.tsx     # 詳細設定タブ（入力パネル）
    ├── SalaryTableSpreadsheet.tsx    # テーブルタブ（表示のみ）
    └── SalaryAdjustmentCriteria.tsx  # 改定基準タブ
```

### 各タブの保存対象

| タブ | コンポーネント | 保存対象 | APIエンドポイント |
|------|---------------|---------|------------------|
| 現基本給設定 | `CurrentSalaryTab` | 従業員の現基本給 | `PUT /api/salary-tables/{id}/current-salaries` |
| 詳細設定 | `SalaryTableInputPanel` | 号俸テーブルパラメータ | `PUT /api/salary-tables/{id}` |
| テーブル | `SalaryTableSpreadsheet` | なし（表示のみ） | - |
| 改定基準 | `SalaryAdjustmentCriteria` | 改定ルール | `POST /api/salary-tables/{id}/adjustment-rules` |

### 現状の問題点

1. 各タブに個別の保存ボタンがあり、UIが統一されていない
2. ユーザーが複数タブを編集した場合、各タブで個別に保存が必要
3. 未保存の変更がある状態でタブを切り替えると、変更が失われる可能性

## 目標

1. ページ右上に統一された保存ボタンを1つだけ配置
2. どのタブで変更があっても、1つのボタンで全て保存
3. 変更の有無を視覚的に表示（バッジ等）
4. 未保存の変更がある場合、ページ離脱時に警告

## 設計

### 1. 状態管理の集約

親コンポーネント（`page.tsx`）で全タブの変更状態を管理する。

```typescript
interface ChangeState {

    data: Map<string, number>  // employeeId -> currentSalary
  }
  settings: {

  }
  adjustmentRules: {

// 親コンポーネントの状態
const [changeState, setChangeState] = useState<ChangeState>({

  adjustmentRules: { hasChanges: false, data: [] },
})
```

### 4. 一括保存処理

```typescript
const handleSaveAll = async () => {
  setIsSaving(true)
  const errors: string[] = []

  try {
    // 1. 現基本給の保存
    if (changeState.currentSalary.hasChanges) {
      const salaries = Array.from(changeState.currentSalary.data.entries())
        .map(([employeeId, currentSalary]) => ({ employeeId, currentSalary }))
      
      const res = await fetch(`/api/salary-tables/${salaryTableId}/current-salaries`, {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ salaries }),
      })
      if (!res.ok) errors.push("現基本給の保存に失敗しました")
    }

    // 2. 詳細設定の保存
    if (changeState.settings.hasChanges && changeState.settings.data) {
      const res = await fetch(`/api/salary-tables/${salaryTableId}`, {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(changeState.settings.data),
      })
      if (!res.ok) errors.push("詳細設定の保存に失敗しました")
    }

    // 3. 改定ルールの保存
    if (changeState.adjustmentRules.hasChanges) {
      const res = await fetch(`/api/salary-tables/${salaryTableId}/adjustment-rules`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ rules: changeState.adjustmentRules.data }),
      })
      if (!res.ok) errors.push("改定ルールの保存に失敗しました")
    }

    // 結果処理
    if (errors.length > 0) {
      toast.error(`一部の保存に失敗しました: ${errors.join(", ")}`)
    } else {
      toast.success("全ての変更を保存しました")
      // 変更状態をリセット
      setChangeState({
        currentSalary: { hasChanges: false, data: new Map() },
        settings: { hasChanges: false, data: null },
        adjustmentRules: { hasChanges: false, data: [] },
      })
      // クエリを再取得
      queryClient.invalidateQueries({ queryKey: ["current-salaries"] })
      queryClient.invalidateQueries({ queryKey: ["salary-tables"] })
      queryClient.invalidateQueries({ queryKey: ["adjustmentRules"] })
    }
  } finally {
    setIsSaving(false)
  }
}
```

### 5. ページ離脱時の警告

```typescript
// useEffect for beforeunload
useEffect(() => {
  const hasAnyChanges = 
    changeState.currentSalary.hasChanges ||
    changeState.settings.hasChanges ||
    changeState.adjustmentRules.hasChanges

  const handleBeforeUnload = (e: BeforeUnloadEvent) => {
    if (hasAnyChanges) {
      e.preventDefault()
      e.returnValue = ""
    }
  }

  window.addEventListener("beforeunload", handleBeforeUnload)
  return () => window.removeEventListener("beforeunload", handleBeforeUnload)
}, [changeState])
```

## 実装手順

### Phase 1: 親コンポーネントの変更（page.tsx）

1. `ChangeState` 型と状態を追加
2. 各子コンポーネントへの `onChange` コールバックを定義
3. 統一保存ボタンのUI追加
4. `handleSaveAll` 関数を実装
5. `beforeunload` イベントハンドラを追加

### Phase 2: CurrentSalaryTab の変更

1. `onChanges` プロップを追加
2. 内部の `editedSalaries` 変更時に `onChanges` を呼び出し
3. 内部の保存ボタンとロジックを削除
4. `saveMutation` を削除（親で処理）

### Phase 3: SalaryTableInputPanel の変更

1. `onChanges` プロップを追加
2. フォーム値変更時に `onChanges` を呼び出し
3. 保存ボタンのUIを削除（計算結果プレビューは維持）
4. `onSubmit` の処理を `onChanges` に統合

### Phase 4: SalaryAdjustmentCriteria の変更

1. `onRulesChange` プロップを追加
2. ルール変更時に `onRulesChange` を呼び出し
3. 内部の保存ボタンとミューテーションを削除

### Phase 5: テスト・検証

1. 各タブで変更を行い、保存ボタンの状態確認
2. 複数タブで変更後、一括保存の動作確認
3. 部分的に保存失敗した場合のエラー表示確認
4. ページ離脱時の警告表示確認

## UI/UX要件

### 保存ボタン

- 位置: ページヘッダー右上（タブの上）
- 状態表示:
  - 変更なし: 非活性（disabled）
  - 変更あり: 活性 + バッジで変更件数表示
  - 保存中: ローディングスピナー表示

### 変更インジケーター

- タブにも変更有無を表示（オプション）
  - タブ名の横にドット（●）を表示
  - 例: `現基本給設定 ●`

### 保存結果通知

- 成功: 緑色トースト「全ての変更を保存しました」
- 部分失敗: 黄色トースト「一部の保存に失敗しました: [詳細]」
- 全失敗: 赤色トースト「保存に失敗しました」

## 注意事項

1. **詳細設定タブの特殊性**: 号俸テーブルパラメータの変更は、テーブル全体の再生成を伴う。既存の確認ダイアログ（`showConfirmDialog`）の処理を維持する必要がある。

2. **改定基準タブの初期データ**: 改定ルールは初期状態でデフォルト値が計算される。変更検知は「保存されている値」と「現在の値」の差分で判定。

3. **テーブルタブ**: 変更対象がないため、変更追跡の対象外。

4. **エラーハンドリング**: 一括保存で一部が失敗した場合、成功した部分の変更状態はリセットし、失敗した部分は保持する。
