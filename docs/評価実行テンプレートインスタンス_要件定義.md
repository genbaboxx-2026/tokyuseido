# 評価実行テンプレートインスタンス 要件定義書

## 1. 概要

### 1.1 背景と課題

現状、評価実行（`/companies/[companyId]/operations/[periodId]`）では、評価テンプレートの内容が見えない状態で従業員の評価を進めている。テンプレートの確認には評価制度設定ページに移動する必要がある。

また、評価テンプレートはマスタとして1つだけ存在するが、実際の運用では「等級×職種ごとに一部項目を変えたい」「今期だけ特定の項目を追加したい」といったニーズがあり、マスタを直接変更すると次回以降の評価にも影響してしまう。

### 1.2 解決方針

評価実行時に「実行用テンプレートインスタンス」を作成する仕組みを導入する。

```
マスタテンプレート（評価制度設定）
    │
    │ 評価開始時にコピー
    ▼
実行用テンプレートインスタンス（評価実行内で編集可能）
    │
    │ 「反映」ボタンで適用
    ▼
各従業員の評価項目（等級×職種が一致する対象に反映）
```

**重要な制約: 実行用インスタンスの編集はマスタテンプレートに一切影響しない。**

---

## 2. 対象範囲

| 評価種別 | 対象 |
|---------|------|
| 360度評価 | `Evaluation360Template` → 実行用インスタンス → 各従業員の360度評価項目 |
| 個別評価 | `EvaluationTemplate`（等級×職種単位） → 実行用インスタンス → 各従業員の個別評価項目 |

---

## 3. 用語定義

| 用語 | 意味 |
|------|------|
| マスタテンプレート | 評価制度設定で管理する元のテンプレート。変更しない限り全期間で共通 |
| 実行用インスタンス | 評価期間ごとにマスタからコピーされたテンプレート。評価実行内で自由に編集可能 |
| 反映 | 実行用インスタンスの内容を、対象の等級×職種に該当する従業員の評価項目に一括適用する操作 |
| オーバーライド | 等級×職種単位で実行用インスタンスの項目を変更すること |

## 4. 機能要件

### 4.1 評価開始時のテンプレートコピー

**タイミング:** 評価期間を開始する時（`EvaluationStartDialog` で開始ボタン押下時）

**360度評価:**
1. 選択された `Evaluation360Template`（マスタ）の内容を丸ごとコピー
2. コピー先: `PeriodEvaluation360Template`（新テーブル）
3. カテゴリ・項目・対象等級・対象職種すべてコピー
4. `sourceTemplateId` でコピー元のマスタテンプレートIDを保持

**個別評価:**
1. 対象の等級×職種に紐づく `EvaluationTemplate`（マスタ）の内容をコピー
2. コピー先: `PeriodEvaluationTemplate`（新テーブル）
3. 各等級×職種ごとに独立したインスタンスを作成
4. `sourceTemplateId` でコピー元のマスタテンプレートIDを保持

**前回の評価期間からのコピー:**
- 評価開始時に「前回の評価期間のインスタンスを引き継ぐ」オプションを提供
- マスタではなく、前回の実行用インスタンスをコピー元として使用可能

### 4.3 実行用インスタンスの編集

#### 4.3.1 360度評価テンプレートの編集

**編集対象:**
- カテゴリの追加・削除・名称変更・並び順変更
- 項目の追加・削除・内容変更・配点変更・並び順変更
- 対象等級・対象職種の変更

**編集UI:**
- テンプレート表示エリアの「編集」ボタンをクリック → 編集モーダル/インライン編集
- 現在の評価制度設定ページのテンプレート編集UIと同じ操作感

**制約:**
- マスタテンプレートには影響しない
- 既に配布済み（ステータスが「配布・回収」以降）の場合は警告を表示

#### 4.3.2 個別評価テンプレートの編集

**編集対象:**
- 等級×職種単位で項目の追加・削除・内容変更・配点変更
- カテゴリの変更

**編集UI:**
- 各等級×職種の「編集」ボタン → 編集モーダル
- 等級×職種ごとに独立して編集可能

**制約:**
- マスタテンプレートには影響しない
- 既に配布済みの従業員がいる場合は警告を表示

### 4.4 従業員への反映

#### 4.4.1 反映のトリガー

**自動反映（デフォルト）:**
- 評価開始時のテンプレートコピー後、対象従業員に自動で初期項目を反映

**手動反映:**
- テンプレート編集後、「全従業員に反映」ボタンで一括反映
- 対象: そのテンプレートの等級×職種に該当する従業員のみ

#### 4.4.2 反映のロジック

```
反映処理:
1. 実行用インスタンスの項目一覧を取得
2. 対象の等級×職種に該当する従業員を抽出
3. 各従業員の評価項目を更新
   - 未回答の項目: テンプレートの内容で上書き
   - 回答済みの項目: 警告を出して確認を求める
     → 「上書き」: テンプレートの内容で置換（回答はリセット）
     → 「スキップ」: 既存の項目を維持
```

#### 4.4.3 個人単位のカスタマイズとの共存

- テンプレートから反映された後、個人単位でさらにカスタマイズ可能（既存機能）
- 個人カスタマイズ済みの項目に対してテンプレートを再反映する場合は警告を表示
- 従業員リストの「変更」列で個人カスタマイズの有無を表示

## 5. データモデル

## 6. API設計

### 6.1 評価期間用360度テンプレート

| メソッド | エンドポイント | 説明 |
|---------|--------------|------|
| GET | `/api/companies/[companyId]/operations/[periodId]/360/template` | 期間用360度テンプレートを取得 |
| PUT | `/api/companies/[companyId]/operations/[periodId]/360/template` | 期間用360度テンプレートを更新 |
| POST | `/api/companies/[companyId]/operations/[periodId]/360/template/apply` | テンプレートを対象従業員に反映 |

### 6.2 評価期間用個別テンプレート

| メソッド | エンドポイント | 説明 |
|---------|--------------|------|
| GET | `/api/companies/[companyId]/operations/[periodId]/individual/templates` | 期間用個別テンプレート一覧（等級×職種ごと） |
| GET | `/api/companies/[companyId]/operations/[periodId]/individual/templates/[templateId]` | 単一テンプレート取得 |
| PUT | `/api/companies/[companyId]/operations/[periodId]/individual/templates/[templateId]` | テンプレート更新 |
| POST | `/api/companies/[companyId]/operations/[periodId]/individual/templates/apply` | テンプレートを対象従業員に一括反映 |
| POST | `/api/companies/[companyId]/operations/[periodId]/individual/templates/[templateId]/apply` | 特定の等級×職種のテンプレートのみ反映 |

### 6.3 評価開始時のコピー

| メソッド | エンドポイント | 説明 |
|---------|--------------|------|
| POST | `/api/evaluations/start` | 既存エンドポイントを拡張。テンプレートコピーを含む |

**リクエストボディの拡張:**

```typescript
interface StartEvaluationRequest {
  companyId: string
  periodId: string
  evaluationTypes: ("360" | "individual")[]
  // 新規追加
  templateSource: "master" | "previous_period"
  previousPeriodId?: string  // templateSource が "previous_period" の場合
}
```

## 8. 処理フロー

### 8.1 評価開始フロー

```
1. ユーザーが評価開始ダイアログで設定を入力
2. テンプレートソースを選択（マスタ or 前回期間）
3. 「開始」ボタン押下

[バックエンド処理]
4. EvaluationPeriod を作成/更新（ステータス: STARTED）
5. テンプレートコピー:
   a. マスタからの場合:
      - 360度: Evaluation360Template → PeriodEvaluation360Template にコピー
      - 個別: 対象の EvaluationTemplate → PeriodEvaluationTemplate にコピー（等級×職種ごと）
   b. 前回期間からの場合:
      - 360度: 前回の PeriodEvaluation360Template → 新しい PeriodEvaluation360Template にコピー
      - 個別: 前回の PeriodEvaluationTemplate → 新しい PeriodEvaluationTemplate にコピー
6. 各従業員に初期評価項目を反映
7. EmployeeEvaluation / Evaluation360Record を作成（periodTemplateId をセット）
```

### 8.2 テンプレート編集フロー

```
1. ユーザーがテンプレートパネルの「編集」ボタンをクリック
2. 編集モーダルが開き、PeriodEvaluation(360)Template の内容を表示
3. 項目の追加・削除・変更を行う
4. 「保存」ボタンで PeriodEvaluation(360)Template を更新
5. ※この時点では従業員の評価項目は変更されない
```

### 8.3 従業員への反映フロー

```
1. ユーザーが「反映」ボタンをクリック
2. 確認ダイアログ表示:
   - 対象従業員数
   - 回答済みの従業員がいる場合は警告
3. 「反映する」で処理開始

[バックエンド処理]
4. 対象の等級×職種に該当する従業員を抽出
5. 各従業員ごとに:
   a. 個人カスタマイズ済み（isCustomized = true）→ スキップ or 上書き（オプション）
   b. 未カスタマイズ → テンプレートの内容で評価項目を更新
6. 処理結果を返却（反映件数、スキップ件数）
```

## 10. 注意事項・制約

- **マスタは読み取り専用**: 評価実行からマスタテンプレートを変更する経路は存在しない
- **期間用テンプレートは期間に紐づく**: 評価期間が削除されれば、期間用テンプレートも削除される（CASCADE）
- **後方互換**: 既存の評価データ（`EvaluationCustomItem`等）との互換性を維持する。`periodTemplateId` はオプショナルで、既存データはnullのまま
- **パフォーマンス**: テンプレートコピーは1回のトランザクションで行い、従業員数が多い場合はバッチ処理を検討
- **同時編集**: 評価期間中のテンプレート編集は1人ずつ（楽観ロック or updatedAt チェック）
