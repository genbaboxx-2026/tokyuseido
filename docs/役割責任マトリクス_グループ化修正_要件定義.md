# 役割責任マトリクス グループ化修正 要件定義

## 概要

役割責任マトリクスにおける従業員人数表示の不一致と、グループ化時の挙動を修正する。

---

## 現状の問題点

### 1. 人数表示の不一致
- マトリクス一覧で「1名」と表示されているのに、モーダルを開くと「2名」と表示される
- 従業員の等級・職種情報と表示が一致していない

### 2. グループ化時の挙動が不明確
- グループ化した場合の人数カウント、モーダル表示のルールが曖昧

---

## 修正後の仕様

### 基本原則

1. **従業員の人数は、その従業員の実際の等級・職種に基づいてカウントする**
2. **グループ化した場合、役割責任の内容は共通だが、従業員の表示はコンテキストによって変わる**

---

### グループ化していない場合

#### マトリクス一覧
| 項目 | 表示内容 |
|------|----------|
| 人数 | その職種×等級に該当する従業員数 |
| 役割責任 | その職種×等級の役割責任 |

#### モーダル
| 項目 | 表示内容 |
|------|----------|
| 該当者 | その職種×等級に該当する従業員一覧 |
| 役割責任 | その職種×等級の役割責任 |

---

### グループ化した場合

**例**: 「技術専門職群」と「経営管理職群」をグループ化
- 正2等級に「技術専門職群」に1名（佐藤）、「経営管理職群」に1名（加藤）がいる場合

#### マトリクス一覧（両セルとも同じ表示）

| 項目 | 表示内容 |
|------|----------|
| 人数 | **グループ合計**: 2名 |
| 役割責任 | **グループ共通**: 同じ内容を表示 |
| バッジ | 「2職種共通」などのグループ表示 |

```
┌──────────────────────────┬──────────────────────────┐
│    技術専門職群          │    経営管理職群          │
├──────────────────────────┼──────────────────────────┤
│ 正2  2名 🔗2職種共通     │ 正2  2名 🔗2職種共通     │
│ 1. 役割責任内容...       │ 1. 役割責任内容...       │  ← 同じ内容
│ 2. ...                   │ 2. ...                   │
└──────────────────────────┴──────────────────────────┘
```

#### モーダル（クリックした職種のみの従業員を表示）

**「技術専門職群」セルをクリックした場合:**

| 項目 | 表示内容 |
|------|----------|
| モーダルタイトル | **「管理 / 技術専門職群 - 全等級」** ← クリックした職種のみ |
| 該当者 | **技術専門職群のみ**: 佐藤（1名） |
| 役割責任 | **グループ共通**: 同じ内容 |

**「経営管理職群」セルをクリックした場合:**

| 項目 | 表示内容 |
|------|----------|
| モーダルタイトル | **「管理 / 経営管理職群 - 全等級」** ← クリックした職種のみ |
| 該当者 | **経営管理職群のみ**: 加藤（1名） |
| 役割責任 | **グループ共通**: 同じ内容 |

```
┌─────────────────────────────────────────────────────────┐
│  役割責任                                      [保存] × │
│  管理 / 技術専門職群 - 全等級   ← クリックした職種のみ  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ▼ 正2  5項目                                           │
│    該当者: 佐藤（1名）  ← クリックした職種のみ          │
│    責任内容:                                            │
│    1. グループ共通の役割責任...   ← 共通内容            │
│    ...                                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 実装タスク

### 1. RoleMatrix.tsx（マトリクス一覧）の修正

#### 人数カウントの修正
- グループ化していない場合: その職種×等級の従業員数
- グループ化した場合: グループ内全職種×その等級の合計従業員数

```typescript
// 現状（問題あり）
employeeCount = cellData.employees.length

// 修正後
if (group) {
  // グループの場合: グループ内全職種の合計
  const groupJobTypeIds = group.jobTypeIds as string[]
  employeeCount = roles
    .filter(r => r.config.grade.id === gradeId && groupJobTypeIds.includes(r.config.jobType.id))
    .reduce((sum, r) => sum + r.employees.length, 0)
} else {
  // 非グループ: その職種のみ
  employeeCount = cellData.employees.length
}
```

### 2. RoleEditModal.tsx（モーダル）の修正

#### 該当者表示の修正
- モーダルを開く際に、クリックした職種IDを渡す
- グループ化している場合でも、クリックした職種の従業員のみを表示
- 役割責任の内容はグループ共通

```typescript
// モーダルに渡すデータ構造
interface RoleDataForModal {
  configId: string
  roleId: string | null
  gradeName: string
  gradeLevel: number
  jobTypeName: string
  responsibilities: string[]  // グループ共通
  positionNames: string[]
  employees: Employee[]       // クリックした職種のみ
}
```

### 3. 修正対象ファイル

| ファイル | 修正内容 |
|----------|----------|
| `src/components/grades/RoleMatrix.tsx` | グループ時の人数カウント修正、モーダルへの職種ID渡し |
| `src/components/grades/RoleEditModal.tsx` | 該当者のフィルタリング処理追加 |

## 完了条件

- [ ] グループ化していない場合、一覧とモーダルの人数が一致する
- [ ] グループ化した場合、一覧ではグループ合計人数が表示される
- [ ] グループ化した場合、モーダルではクリックした職種の従業員のみ表示される
- [ ] グループ化した場合、モーダルのタイトルはクリックした職種のみ表示（グループ全体ではない）
- [ ] グループ化した場合、役割責任の内容はグループで共通
- [ ] 従業員の等級・職種情報に基づいて正しくカウントされる

---

## テストケース

### ケース1: グループ化なし
- 技術専門職群・正2に従業員A（1名）
- 一覧: 1名
- モーダル: 1名（従業員A）

### ケース2: グループ化あり
- 技術専門職群・正2に従業員A（1名）
- 経営管理職群・正2に従業員B（1名）
- 2つの職種をグループ化
- 一覧（両セル）: 2名
- 技術専門職群モーダル: 1名（従業員Aのみ）
- 経営管理職群モーダル: 1名（従業員Bのみ）
- 役割責任: 両モーダルで同じ内容
